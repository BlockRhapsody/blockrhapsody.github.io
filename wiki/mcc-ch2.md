# 第二章 权限等级


## 2.1 基岩版权限等级概述

基岩版的权限等级共分为五个级别，数字越大权限越高：

| 等级 | 名称 | 说明 |
| :--- | :--- | :--- |
| 0级 | Normal（普通） | 访客和成员，只能执行聊天、查看帮助等基础命令 |
| 1级 | Operator（操作员） | 大多数指令的权限等级要求 |
| 2级 | Host（主机/房主） | 可执行`/op`、`/deop`、`/reload`等管理指令 |
| 3级 | Automation（自动化） | 主要用于`/setmaxplayers`等容量管理指令 |
| 4级 | Owner（所有者） | 最高权限，可执行`/stop`、`/save`、`/whitelist`等 |


## 2.2 不同场景下的权限等级

| 执行环境 | 权限等级 |
| :--- | :--- |
| 命令方块和命令方块矿车 | 1级 |
| 服务器控制台 | 4级 |
| 函数或附加包中的脚本 | 1级 |
| `/execute`执行子命令 | 1级 |
| 玩家开启"操作员命令"（服务器） | 1级 |
| 玩家权限为管理员 | 2级 |
| 单人游戏开启作弊 | 3级 |

### 哪些指令必须1级以上？

绝大部分修改游戏世界的指令都需要1级或以上权限：
- `/give` — 给予物品
- `/tp` — 传送实体
- `/gamemode` — 修改游戏模式
- `/kill` — 清除实体
- `/time` — 修改时间
- `/weather` — 修改天气
- `/kick` — 踢出玩家

### 哪些指令仅限于2级到4级？

- `/op`和`/deop` — 2级
- `/reload` — 2级
- `/setmaxplayers` — 3级
- `/stop` — 4级
- `/save` — 4级
- `/whitelist` — 4级
- `/permission` — 4级


## 2.3 玩家权限

### 访客

不能放置或破坏方块，不能攻击实体，不能与任何方块交互。权限等级：0级。

### 成员

可以正常进行游戏。默认权限等级为0级。在开启作弊的世界中打开"操作员命令"开关后：
- 服务器环境：提升为1级
- 单人/局域网环境：提升为3级

### 管理员

拥有执行服务器管理指令的能力。权限等级：2级。

### 自定义

由下方选项控制：建造、挖掘、使用门和开关、打开容器、攻击玩家、攻击生物、操作员命令、传送。

> ⚠️ **关键提示**："访客""成员""管理员"是游戏界面中显示的身份标签，而"0级到4级"是游戏引擎实际用于判断指令能否执行的数字。两者相互关联，但不能混为一谈。

> 值得注意的是：网易联机大厅（不包括本地联机）和服务器内，玩家都无法获得3级和4级的权限等级，但可以使用"魔法指令"以4级权限等级执行命令。
