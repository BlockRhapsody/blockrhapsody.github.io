# 第三章 指令的构成、解析与执行


## 3.1 指令的语法结构

一条完整的指令由若干以空格隔开的片段组成。

### 指令名

指令名是指令的第一个片段，例如`/give @s diamond 64`中的`give`就是指令名。在基岩版中，指令名不区分大小写。

### 参数

参数类型包括：
- `target`（目标选择器）：如`@s`、`@a`、`@e[type=zombie]`
- `int`（整数）：如`64`、`1`、`-5`
- `float`（浮点数）：如`3.14`、`0.5`
- `string`（字符串）：文本内容
- `message`（消息文本）：一段文字
- `xyz`（坐标）：三个浮点数
- `Block`（方块ID）：如`minecraft:diamond_block`
- `Item`（物品ID）：如`minecraft:diamond_sword`
- `json`（JSON文本）：结构化文本

### 可选项与必选项的标识

- **尖括号`<>`**：必选参数
- **方括号`[]`**：可选参数
- **无括号的字面量**：固定内容，必须原样输入

### 空格分隔规则

指令的各个片段之间必须用空格分隔。如果参数本身包含空格，需要用英文引号包裹。


## 3.2 指令的解析过程

### 客户端解析

- 语法提示：输入时按`Tab`键可查看可用指令
- 参数预览：参数可解析时，语法提示从灰色变为白色

### 服务端解析

- 指令识别：根据指令名查找对应的指令处理器
- 参数校验：验证参数类型是否正确、值是否有效
- 权限检查：验证执行者是否具有所需权限

### 常见解析错误及原因

1. "未知的命令"：指令名拼写错误或不存在
2. "语法错误"：参数数量不对、类型不匹配
3. "你没有权限执行此命令"：权限等级不足
4. 参数值无效：输入的ID不存在


## 3.3 指令的执行流程

### 执行结果的反馈

- **无法解析**：语法错误，成功次数为0
- **执行失败**：语法正确但未产生预期效果，成功次数为0
- **执行成功**：至少完成了一次有效操作，成功次数大于0

### 指令的成功次数在链式命令中的用途

- **红石比较器检测**：成功次数可通过红石比较器输出信号强度
- **连锁命令方块的条件执行**："有条件"模式下，只有前一个命令执行成功才触发
- **`/execute store`存储**：将成功次数存储到计分板


## 3.4 指令的调试方法

### 如何通过报错信息定位问题

- **红色文本提示**：聊天栏中的错误信息
- **命令方块输出框**：命令方块界面下方的输出区域
- **内容日志**：在创作者设置中开启，记录详细错误信息

> 报错信息只会报错一个错误内容，通常是指令字符串的第一个错误内容。

### 分步测试的技巧

1. 在聊天栏中逐条验证
2. 使用`/execute if`子命令进行条件测试
3. 逐层构建命令链
4. 利用计分板记录中间状态，或使用`/say`汇报


## 3.5 指令术语

| 术语 | 含义 |
| :--- | :--- |
| 命令 | 通过文本形式向游戏发出的操作请求 |
| 斜杠`/` | 命令的前缀符号 |
| 命令名 | 命令的第一个片段 |
| 参数 | 命令名之后的片段 |
| 字面量 | 命令中固定不变的部分 |
| 目标选择器 | 选中实体的特殊语法 |
| 执行者 | 命令的实际发起主体 |
| 执行原点 | 命令执行时的参考位置 |
| 权限等级 | 控制命令执行权限的数字体系 |
| 解析 | 将文本命令转换为可执行指令的过程 |
| 命令方块 | 自动执行命令的特殊方块 |
| 函数 | `.mcfunction`文件中存储的命令集合 |
| 游戏刻 | 游戏循环的最小单位，目标频率每秒20次 |
